# Security Policy

## Supported Versions

Security fixes are provided for the latest released version of Backlot.

## Reporting A Vulnerability

Please do not open a public issue for suspected private-data leaks,
encryption failures, or vulnerabilities that could expose user archives.

Report vulnerabilities through GitHub private vulnerability reporting for this
repository. Include:

- the Backlot version or commit;
- your operating system;
- the commands you ran;
- whether archive encryption or auto-sync was enabled;
- the smallest reproduction you can share safely.

Expected response:

- initial acknowledgement when the report is received;
- follow-up questions if reproduction details are missing;
- a fix or mitigation plan when the issue is confirmed;
- public disclosure after a fix is available, when appropriate.
